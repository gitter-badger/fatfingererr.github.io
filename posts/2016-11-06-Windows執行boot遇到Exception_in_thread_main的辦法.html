Windows 執行 boot 遇到 Exception in thread main 的辦法

2016年11月6日


由於在 Windows 中 boot.exe 是以一個執行檔的方式存在

如果你沒有把他移到 C:\Windows 中很有可能不小心把它弄不見

導致你在 cmd 無法執行 boot 指令，但是因為你曾經執行過 boot

所以你的使用者資料夾中還是有一個資料夾 .boot 此時如果你又重新去下載了一次 boot.exe

有時候會發生執行失敗，得到如下的錯誤指令：


Exception in thread "main" java.lang.reflect.InvocationTargetException
...
...


此時你只需要移除掉 .boot 的資料夾（可能會花一些時間）

再重新把 boot.exe 放進 Windows 資料夾

並在打算執行 boot 的專案目錄下，執行 【兩次】 boot dev 指令就可以了：

cd C:\Users\ [User]
rm rf .boot
cd [project]
boot dev
... 開始重新下載
boot dev
... 正式執行


本篇參考boot-clj/boot 中 2016/5 回報的 Issue brew version does not work #467 的解決方法


如果你有遇到這個問題，但不是這樣的解法，也歡迎分享哦！
